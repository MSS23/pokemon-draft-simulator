Create `src/components/feedback/FloatingFeedbackButton.tsx` — a client component that renders a fixed-position button in the bottom-right corner linking to /feedback.

```typescript
'use client'

import Link from 'next/link'
import { usePathname } from 'next/navigation'
import { MessageSquare } from 'lucide-react'

/**
 * Floating feedback button — visible on all pages except /feedback itself.
 * Positioned bottom-right, above the fold, does not obstruct content.
 * Links to the existing /feedback page (Discord webhook integration).
 */
export function FloatingFeedbackButton() {
  const pathname = usePathname()

  // Hide on the feedback page itself to avoid redundancy
  if (pathname === '/feedback') return null

  return (
    <Link
      href="/feedback"
      className="fixed bottom-6 right-6 z-50 flex items-center gap-2 rounded-full bg-primary px-4 py-2.5 text-sm font-medium text-primary-foreground shadow-lg transition-all hover:bg-primary/90 hover:shadow-xl hover:scale-105 active:scale-95 print:hidden"
      aria-label="Send feedback"
    >
      <MessageSquare className="h-4 w-4" />
      <span className="hidden sm:inline">Feedback</span>
    </Link>
  )
}
```

Design decisions:
- **Passive link, not a modal** — per context notes, the button navigates to the existing feedback page rather than opening an inline form. This keeps the component tiny and reuses the full feedback form.
- **Hide on /feedback page** — no duplicate CTA when user is already there.
- **Mobile-friendly** — on small screens, only shows the icon (no text label) to save space. Uses `hidden sm:inline` for the text.
- **z-50** — high z-index to stay above page content but below modals (which use z-[9999]).
- **bottom-6 right-6** — positioned to not overlap with the Toaster (top-center) or any bottom navigation.
- **print:hidden** — don't show in print views.
- Uses existing `bg-primary` (the #dc2855 brand color) for consistency.
- `lucide-react` MessageSquare icon matches the feedback page's icon usage.
