Create `tests/load/draft-load-test.js` as a k6 script. k6 uses vanilla ES module syntax with k6-specific imports — NOT Node.js or browser APIs.

The script must use environment variables for configuration (never hardcode credentials):
- `__ENV.SUPABASE_URL` — e.g. https://abcdef.supabase.co
- `__ENV.SUPABASE_ANON_KEY` — anonymous key
- `__ENV.DRAFT_ID` — UUID of a pre-created draft in the target environment
- `__ENV.BASE_URL` — app base URL (e.g. https://draftpokemon.com), used for page load checks

Structure the script with these sections:

**1. Options block** — defines the load profile and thresholds:
```javascript
export const options = {
  scenarios: {
    draft_picks: {
      executor: 'constant-vus',
      vus: 8,           // 8 concurrent players
      duration: '60s',  // 60-second sustained load
    },
  },
  thresholds: {
    // PERF-04 success criteria:
    'http_req_duration{scenario:draft_picks}': ['p(95)<500'],  // p95 under 500ms
    'http_req_failed{scenario:draft_picks}': ['rate<0.01'],    // <1% error rate
    // Named metric for pick latency specifically:
    'pick_latency': ['p(95)<500'],
  },
}
```

**2. Setup function** — validates env vars are set (k6 `setup()` runs once before VUs start):
```javascript
export function setup() {
  const required = ['SUPABASE_URL', 'SUPABASE_ANON_KEY', 'DRAFT_ID']
  for (const key of required) {
    if (!__ENV[key]) {
      throw new Error(`Missing required env var: ${key}. See tests/load/README.md`)
    }
  }
  return { supabaseUrl: __ENV.SUPABASE_URL, anonKey: __ENV.SUPABASE_ANON_KEY, draftId: __ENV.DRAFT_ID }
}
```

**3. Default function** — the per-VU scenario. Each VU simulates one player:
- Uses `k6/http` for HTTP calls
- Uses `k6/metrics` Trend for `pick_latency`
- Uses `k6/execution` to get `vu.idInTest` for team selection
- Picks a team based on VU id (vu.idInTest % 8, assuming 8 teams in the test draft)
- Makes a pick attempt via POST to Supabase RPC:
  ```
  POST ${supabaseUrl}/rest/v1/rpc/make_draft_pick
  ```
- Records latency to the `pick_latency` Trend metric
- Uses `check()` to validate: response status is 200, response body is not an error
- Includes a 1-second sleep between picks (`sleep(1)`) to simulate realistic think time

**4. Pokemon pool** — define a small array of 20 valid Pokemon IDs (use national dex numbers as strings: '1', '4', '7', '25', etc.) to rotate through. Each VU picks the pokemon at index `(iteration * 8 + vuId) % pokemonPool.length` to spread picks across different Pokemon.

**5. Error handling** — if the RPC returns a Supabase error (non-200 status or `{ error: ... }` in body), log the error message and count it in `http_req_failed`. Do not throw — continue to the next iteration.

**6. Comments** — Add a header comment block:
```javascript
/**
 * k6 Load Test: Draft Pick Latency
 * 
 * Tests: 8 concurrent players making picks via Supabase RPC
 * Thresholds: p95 pick latency < 500ms, error rate < 1%
 * 
 * Prerequisites:
 * - A test draft must exist in Supabase with 8 teams and 'active' status
 * - Set env vars: SUPABASE_URL, SUPABASE_ANON_KEY, DRAFT_ID
 * - See tests/load/README.md for full setup instructions
 * 
 * Run: k6 run -e SUPABASE_URL=... -e SUPABASE_ANON_KEY=... -e DRAFT_ID=... tests/load/draft-load-test.js
 */
```

IMPORTANT: Use only k6 built-in imports:
- `import http from 'k6/http'`
- `import { sleep, check } from 'k6'`

- `import { vu } from 'k6/execution'`

Do NOT use Node.js `require()`, `process.env` (use `__ENV`), `console.log` (use `console.log` — k6 supports it), or browser APIs.
