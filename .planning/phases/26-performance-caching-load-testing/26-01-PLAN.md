<objective>
Add a CDN-cacheable PokeAPI proxy route so Pokemon data is served from Vercel's Edge Network on repeat requests, and bump TanStack Query stale times for static Pokemon data to 30 minutes so in-flight drafts never re-fetch already-loaded data.

Purpose: Pokemon data is entirely static — species don't change. Currently every user fetches directly from pokeapi.co with no shared CDN cache, burning bandwidth on redundant network calls. A proxy with `s-maxage=86400` lets Vercel cache responses globally. Separately, a 5-minute staleTime on Pokemon list queries means an 8-player draft can trigger 8x redundant fetches during a single session. 30-minute staleTime eliminates this.

Output: `/api/pokemon/[id]` proxy route with CDN headers, updated staleTime in `usePokemon.ts` for all static-data hooks.
</objective>

  <action>
Create `src/app/api/pokemon/[id]/route.ts` as a Next.js App Router GET handler. This is a thin proxy — it fetches the raw PokeAPI JSON and returns it with CDN cache headers.

Implementation requirements:
1. Extract `id` from `params` using `{ params }: { params: Promise<{ id: string }> }` (Next.js 15 dynamic params are async).
2. Validate `id`: must be non-empty string, max 50 chars, alphanumeric + hyphens only. Return 400 if invalid.
3. Fetch `https://pokeapi.co/api/v2/pokemon/${id}` with a 10-second AbortController timeout. If PokeAPI returns non-2xx, forward the same status code (404 from PokeAPI → 404 from our proxy).
4. Return the raw JSON body with these response headers:
   - `Cache-Control: public, s-maxage=86400, stale-while-revalidate=3600`
   - `CDN-Cache-Control: public, s-maxage=86400`
   - `Content-Type: application/json`
5. On fetch error or timeout, return 502 with `{ error: 'PokeAPI unavailable' }` (no Cache-Control on error responses).
6. Do NOT parse or transform the PokeAPI response — return raw JSON so Vercel can cache the bytes directly.

NOTE: This proxy is for CDN caching purposes. The existing `fetchPokemon` in `pokemon-api.ts` calls PokeAPI directly (client-side) and is NOT modified in this task — the staleTime task (Task 2) ensures client-side caching is sufficient. This proxy is the server-side caching layer for future use and for the success criteria verification (s-maxage header on repeat requests).

Do NOT add authentication to this endpoint — Pokemon data is public.
Do NOT add rate limiting to this endpoint in this plan — it's a read-only public proxy for static data.
