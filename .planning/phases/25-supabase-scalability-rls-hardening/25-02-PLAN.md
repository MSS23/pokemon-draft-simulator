**Step 2: Add beforeunload safety net to DraftRealtimeManager**

In `src/lib/draft-realtime.ts`, add a `beforeunload` event listener inside the `subscribe()` method that calls `cleanup()` as a safety net for hard navigations:

In the `subscribe()` method, after `this.channel.subscribe(...)`, add:

```typescript
// Safety net: hard navigation (tab close, browser back) bypasses React cleanup.
// beforeunload ensures the channel is removed from Supabase's subscriber list.
const handleBeforeUnload = () => {
  if (this.channel) {
    // Synchronous untrack — best effort, no await
    this.channel.unsubscribe()
    supabase.removeChannel(this.channel)
    this.channel = null
  }
}
window.addEventListener('beforeunload', handleBeforeUnload)
// Store reference so cleanup() can remove it
this.beforeUnloadHandler = handleBeforeUnload
```

Add `private beforeUnloadHandler: (() => void) | null = null` to the class fields.

In the `cleanup()` method, add before the channel unsubscribe:
```typescript
if (this.beforeUnloadHandler && typeof window !== 'undefined') {
  window.removeEventListener('beforeunload', this.beforeUnloadHandler)
  this.beforeUnloadHandler = null
}
```

This only runs client-side. The `typeof window !== 'undefined'` guard prevents SSR errors.

IMPORTANT CONTEXT: Supabase Realtime WebSocket connections bypass Next.js middleware entirely — they go directly from browser to wss://*.supabase.co. Next.js middleware cannot intercept or rate-limit these connections.

The correct implementation for RATE-05 is to:
1. Add a channel count guard in `DraftRealtimeManager.subscribe()` that rejects if `supabase.getChannels().length` exceeds a per-client threshold
2. Add a new Upstash rate limiter key in middleware for the `/api/` WebSocket upgrade handshake endpoint — but since supabase uses its own WS endpoint, this is about rate-limiting the application-level "join draft" actions that initiate channel creation

**Implement in `src/lib/draft-realtime.ts` (in the subscribe() method, at the top before channel creation):**

```typescript
// RATE-05: Guard against channel accumulation
// Supabase free tier supports 200 concurrent connections; enforce a per-page limit.
// supabase.getChannels() returns all active channels for this client instance.
const activeChannels = supabase.getChannels()
const draftChannels = activeChannels.filter(ch => ch.topic.startsWith('draft:') || ch.topic.startsWith('wishlist_'))
if (draftChannels.length >= 5) {
  // Log for visibility but do not hard-reject — clean up stale channels first
  log.warn(`[RATE-05] Channel count ${draftChannels.length} exceeds threshold. Cleaning up stale channels before subscribing.`)
  const staleChannels = draftChannels.filter(ch =>
    ch.topic !== `draft:${this.draftId}` &&
    ch.topic !== `wishlist_${this.draftId}`
  )
  for (const stale of staleChannels) {
    await stale.unsubscribe()
    supabase.removeChannel(stale)
    log.info(`[RATE-05] Removed stale channel: ${stale.topic}`)
  }
}
```
